# 專題解析

本文整理福庫銀行專案的整體程式架構、模組分工、主要流程、技術使用方式與建議閱讀順序。內容聚焦在程式碼本身與功能設計，不記錄分支比較過程或更新狀態。

## 1. 專案定位

福庫銀行是一套銀行業務系統，提供客戶端與管理端兩套操作介面。

| 使用端 | 使用者 | 前端入口 | 後端 API 型態 | 主要目的 |
|---|---|---|---|---|
| 客戶端 | 一般顧客 | `/`, `/login`, `/register`, `/user/**` | 多數為 `/api/customer/**` 或業務模組 API | 顧客登入、查帳、轉帳、申請帳戶、申請貸款、使用信用卡、接收通知 |
| 管理端 | 行員、管理者、風控或系統角色 | `/admin/login`, `/admin/**` | 多數為 `/api/auth/**`, `/api/admin/**`, `/api/customers/**` | 員工登入、客戶管理、業務審核、交易查詢、風控審核、系統日誌 |

整體業務分成幾個主模組：

| 模組 | 主要職責 |
|---|---|
| `common` | 共用設定、Security、JWT、CORS、例外處理、統一回應、Email、檔案儲存、匯率、Line Pay 設定 |
| `auth` | 管理端登入、員工帳號、角色、部門、系統操作日誌 |
| `customer` | 客戶註冊、登入、Email 驗證、密碼重設、個資、登入紀錄、裝置管理 |
| `account` | 帳戶、開戶申請、轉帳、預約轉帳、常用帳號、換匯、存提款、交易紀錄、電子存摺、跨模組帳務 |
| `loan` | 貸款申請、行員聯繫、二次填單、補件、風控送審、撥款、貸款帳戶、還款 |
| `creditcard` | 卡別、信用卡申請、補件、明細審核、發卡、交易、刷退、帳單、繳款、Line Pay |
| `risk` | 黑名單、信用資訊、風險事件、風控檢查、人工審核任務、callback |
| `notification` | 客戶站內通知、未讀數、標記已讀、通知偏好 |

## 4. 啟動與開發流程

### 4.1 後端啟動

```powershell
.\mvnw.cmd spring-boot:run
```

後端入口是 `FukuBankApplication.java`。

啟動時需要注意：

1. `application.properties` 只匯入 `application-local.properties`。
2. DB、SMTP、JWT secret、Line Pay 等私密設定不應提交。
3. `@EnableAsync` 啟用非同步工作。
4. `@EnableJpaAuditing` 啟用部分 entity 的自動時間欄位。
5. `FukuBankApplication` 內有資料庫還原工具方法，但正式啟動不會自動重置 DB。

### 4.2 前端啟動

```powershell
cd frontend
npm install
npm run dev
```

前端預設 `http://localhost:5173`，後端預設 `http://localhost:8080`。

管理端使用 Session Cookie，開發時前後端都建議使用 `localhost`，避免 `localhost` 和 `127.0.0.1` 混用導致 Cookie domain 不一致。

### 4.3 常用檢查

```powershell
.\mvnw.cmd test
```

```powershell
cd frontend
npm.cmd run test:unit
npm.cmd run build
```

Windows PowerShell 若被 `npm.ps1` 執行原則擋住，可以改用 `npm.cmd run ...`。

## 5. 認證與授權

系統同時存在管理端 Session 與客戶端 JWT。

| 使用端 | 認證方式 | 前端保存 | 後端入口 | 權限依據 |
|---|---|---|---|---|
| 管理端 | Session-based | `localStorage.auth_user` + Cookie | `POST /api/auth/login` | Spring Security Session + role |
| 客戶端 | JWT-based | `localStorage.customer_user`, `localStorage.customer_token` | `POST /api/customer/auth/login` | JWT claim：`role`, `customerId` |

### 5.1 管理端登入流程

```text
AdminLoginView
-> frontend/src/stores/auth.js
-> POST /api/auth/login
-> AuthController
-> AuthenticationManager
-> CustomUserDetailsService
-> AuthEmp / AuthRole
-> HttpSession / SecurityContext
-> /api/auth/me
-> router guard
```

管理端重點檔案：

| 檔案 | 功能 |
|---|---|
| `common/config/SecurityConfig.java` | Session 與 JWT 共用安全設定 |
| `auth/controller/AuthController.java` | 管理端登入、登出、me |
| `auth/service/CustomUserDetailsService.java` | 管理端帳密驗證資料來源 |
| `frontend/src/stores/auth.js` | 管理端登入狀態 |
| `frontend/src/router/index.js` | 管理端路由守衛 |

### 5.2 客戶端登入流程

```text
UserLoginView
-> frontend/src/stores/customerAuth.js
-> POST /api/customer/auth/login
-> CustomerAuthController
-> CustomerAuthServiceImpl
-> CustomerAuth / CustomerProfile
-> JwtUtil
-> localStorage.customer_token
-> axios request interceptor
-> JwtAuthenticationFilter
-> SecurityContext
```

客戶端重點檔案：

| 檔案 | 功能 |
|---|---|
| `customer/controller/CustomerAuthController.java` | 客戶註冊、登入、驗證信、密碼重設 |
| `customer/service/CustomerAuthServiceImpl.java` | 客戶登入與註冊邏輯 |
| `common/util/JwtUtil.java` | JWT 產生與解析 |
| `common/config/JwtAuthenticationFilter.java` | 每次請求解析 JWT |
| `frontend/src/stores/customerAuth.js` | 客戶登入狀態 |
| `frontend/src/api/axios.js` | token、Cookie、錯誤攔截 |

### 5.3 SecurityConfig 重點

| 規則 | 說明 |
|---|---|
| `/api/auth/login`, `/api/auth/logout` | 管理端登入登出公開 |
| `/api/customer/auth/register`, `/login`, `/request-reset`, `/reset-password`, `/verify-email` | 客戶註冊登入與密碼重設公開 |
| `/uploads/**`, `/img/**` | 靜態資源公開 |
| `/api/public/**`, `/api/linepay/**`, `/api/loan-applications/rate-rules` | 公開 API 或支付入口 |
| `/api/risk/**`, `/api/loan-callbacks/**`, `/api/transfer-callbacks/**`, `/api/account-callbacks/**` | 限制本機 IP |
| 其他 API | 需要登入 |

## 7. 後端模組解析

### 7.1 Auth

| 分層 | 代表檔案 | 職責 |
|---|---|---|
| Controller | `AuthController`, `AuthActionLogController` | 管理端登入、登出、me、系統日誌 |
| Service | `AuthEmpService`, `CustomUserDetailsService`, `AuthActionLogService` | 員工、角色、登入驗證、操作紀錄 |
| Entity | `AuthEmp`, `AuthRole`, `AuthDept`, `AuthLoginLog`, `AuthActionLog` | 管理端員工、角色、部門與日誌資料 |
| Frontend | `AdminLoginView`, `EmployeeListView`, `SystemLogView` | 管理端登入、員工管理、系統日誌 |

Auth 的主軸是管理端身分。登入後靠 Session 維持狀態，前端用 `auth.js` 保存使用者資訊，router 依 meta role 控制頁面進入。

### 7.2 Customer

| 分層 | 代表檔案 | 職責 |
|---|---|---|
| Controller | `CustomerAuthController`, `CustomerController`, `CustomerSecurityController` | 客戶註冊登入、個資、裝置、登入紀錄 |
| Service | `CustomerAuthServiceImpl`, `CustomerServiceImpl`, `CustomerSecurityService`, `LoginAuditService` | 帳密驗證、Email 驗證、密碼重設、個資、登入稽核 |
| Entity | `CustomerProfile`, `CustomerAuth`, `CustomerLoginLog`, `CustomerDevice` | 客戶主資料、帳密、登入紀錄、裝置 |
| Frontend | `RegisterView`, `UserLoginView`, `UserProfileView`, `Security*.vue` | 客戶註冊登入、個資與安全設定 |

Customer 的主軸是客戶身分。註冊、登入、Email 驗證、密碼重設都在這裡；Account 開戶審核後也會把部分資料同步回 `CustomerProfile`。

### 7.3 Account

| 分層 | 代表檔案 | 職責 |
|---|---|---|
| Controller | `AccountController`, `CustomerAccountController`, `TransferController`, `ScheduledTransferController`, `FavoriteAccountController`, `CashController`, `PassbookPdfController`, `AccountIntegrationController` | 帳戶查詢、轉帳、預約轉帳、常用帳號、存提款、電子存摺、貸款與信用卡帳務整合 |
| Service | `AccountService`, `TransferService`, `ScheduledTransferService`, `FavoriteAccountService`, `PassbookPdfService`, `AccountIntegrationService` | 帳務規則、交易紀錄、跨模組整合 |
| Entity | `Account`, `TransLog`, `AccountApplication`, `FavoriteAccount`, `ScheduledTransfer`, `PendingTransfer` | 帳戶、交易、開戶申請、常用帳號、預約與待審轉帳 |
| Frontend | `UserAccountsView`, `TransferView`, `ScheduledTransferView`, `FavoriteAccountsView`, `EPassbookView`, `TransLogView` | 客戶帳務功能與後台交易查詢 |

Account 是金流核心。一般轉帳、預約轉帳、貸款撥款、貸款還款、信用卡繳款都會落到帳務交易與帳戶餘額。電子存摺 PDF 由 `PassbookPdfService` 產生，並透過 Common 的 `PdfCjkFontLoader` 載入共用中文字型，避免不同 PDF 服務各自處理字型路徑。

### 7.4 Loan

| 分層 | 代表檔案 | 職責 |
|---|---|---|
| Controller | `LoanApplicationController`, `LoanAdminController`, `LoanDocumentController`, `LoanAccountController`, `LoanAccountAdminController`, `LoanCallbackController` | 客戶申請、行員審核、補件、貸款帳戶、callback |
| Service | `LoanApplicationService`, `LoanDocumentService`, `LoanAccountService`, `LoanRepaymentService`, `LoanContractPdfService` | 申請生命週期、補件、撥款後建帳、還款、契約 PDF |
| Client / Scheduler | `LoanRiskClient`, `LoanRepaymentScheduler` | 風控送審、補件通知、逾期與到期提醒 |
| Entity | `LoanApplication`, `LoanReviewDetail`, `LoanContactLog`, `LoanDocument`, `LoanAccount`, `LoanRepayment` | 貸款申請、審核、聯繫、文件、帳戶、還款期數 |
| Frontend | `LoanApplyView`, `LoanStatusView`, `LoanAccountView`, `LoanRepaymentView`, `LoanApplicationView`, `LoanAccountAdminView` | 客戶申請與還款、行員審核與帳戶管理 |

Loan 是跨模組流程。它會串 Customer 取客戶資料，串 Risk 做審核，串 Account 做撥款與還款，串 Email / Notification 做通知。貸款核准並完成撥款後，`LoanApplicationService` 會產生貸款契約 PDF，並透過 `EmailService.sendLoanContractEmail` 搭配 `templates/mail/loan-contract.html` 寄出附件。若要快速追貸款功能，建議先看 `LoanApplicationController`、`LoanAdminController`、`LoanDocumentController`、`LoanAccountController` 這幾個 API 入口，再回到 `LoanApplicationService` 與各 service 方法。

### 7.5 Credit Card

| 分層 | 代表檔案 | 職責 |
|---|---|---|
| Controller | `CardApplicationController`, `CardApplicationAdminController`, `CardController`, `CardTxnController`, `CardBillController`, `CardPaymentController`, `LinePayController` | 卡別、申請、補件、審核、發卡、交易、帳單、繳款、Line Pay |
| Service | `CardAppService`, `CardReviewService`, `CreditCardService`, `CardTxnService`, `BillService`, `CardPaymentService`, `LinePayService` | 信用卡申請、審核、交易、帳單、繳款與支付整合 |
| Entity | `CardType`, `CardApplication`, `CardApplicationItem`, `CreditCard`, `CardTransaction`, `CardBill`, `Merchant` | 信用卡主資料、申請、明細、交易、帳單、商店 |
| Mapper | `creditcard/mapper/*` | MapStruct DTO / Entity 轉換 |
| Frontend | `CardApplicationForm`, `CardView`, `CardTxnView`, `CardBillView`, `CardApplicationList`, `CardBillDetailView` | 客戶申請與查詢、管理端審核與帳務 |

Credit Card 是專案中使用 MapStruct 最多的模組。閱讀時可以把 mapper 當成 DTO 與 entity 之間的翻譯層。信用卡帳單也有 PDF 與 Email 附件情境，閱讀時可一起留意 `CardBillPDFService`、帳單信件模板與 PDF 字型處理方式。

### 7.6 Risk

| 分層 | 代表檔案 | 職責 |
|---|---|---|
| Controller | `RiskCheckController`, `RiskReviewController`, `ReviewTaskController`, `RiskEventController`, `BlackListController`, `CreditScoreController` | 風控檢查、人工審核、任務、事件、黑名單、信用資訊 |
| Service | `RiskCheckService`, `RiskReviewService`, `ReviewTaskService`, `RiskEventService`, `CallbackService`, `BlackListService`, `CreditScoreService` | 風險判斷、事件紀錄、審核任務、callback |
| Entity | `RiskEventLog`, `ReviewTask`, `Blacklist`, `CustomerCreditInfo` | 風控事件、審核任務、黑名單、信用資料 |
| Frontend | `RiskEventView`, `ReviewTask`, `BlackListView`, `CreditList`, `CreditDetail` | 風控事件與人工審核管理 |

Risk 是跨模組守門員。登入、轉帳、貸款等流程都可能呼叫風控，風控決策可直接通過、拒絕或轉人工審核。

### 7.7 Notification

| 分層 | 代表檔案 | 職責 |
|---|---|---|
| Controller | `NotificationController`, `NotificationPreferenceController` | 通知列表、未讀數、標記已讀、偏好設定 |
| Service | `NotificationServiceImpl`, `NotificationPreferenceServiceImpl` | 建立通知、查詢通知、更新偏好 |
| Entity | `Notification`, `NotificationPreference` | 通知資料與偏好設定 |
| Frontend | `UserLayout`, `NotificationSettingsView` | 通知鈴鐺、未讀狀態、偏好設定 |

Notification 讓業務事件能回到客戶端。它可和 Email 並行，一個做站內提醒，一個做信件通知。

## 8. 前端結構解析

### 8.1 Layout

| 檔案 | 功能 |
|---|---|
| `frontend/src/layouts/UserLayout.vue` | 客戶端主框架、側邊選單、通知鈴鐺、登入者資訊 |

### 8.2 Router

`frontend/src/router/index.js` 負責：

1. 定義首頁、登入、註冊、客戶端頁面、管理端頁面。
2. 設定路由 meta，例如是否需要登入、角色限制、layout。
3. 客戶端使用 `customerAuth` 判斷 JWT 登入狀態。
4. 管理端使用 `auth` store 與 `/api/auth/me` 確認 session。
5. 未授權時導向登入頁或錯誤頁。

### 8.3 Store

| 檔案 | 功能 |
|---|---|
| `frontend/src/stores/auth.js` | 管理端登入狀態 |
| `frontend/src/stores/customerAuth.js` | 客戶端登入狀態與 token |
| `frontend/src/stores/counter.js` | Vue scaffold 保留或簡單狀態範例 |

### 8.4 API Layer

## 13. 閱讀提醒

1. 專案實際入口是 `FukuBankApplication.java`。
2. Java 21 編譯時要使用目前的 Lombok `1.18.38`。
3. `application-local.properties` 是本機私密設定的核心，不應提交。
4. 管理端 Session 和客戶端 JWT 共用 Security 設定，debug 權限時要先分清楚是哪一種登入。
5. 信用卡客戶端 API 路徑多為 `/user/card-*`，和多數 `/api/**` API 命名風格不同。
6. 風控與 callback API 有 IP 限制，本機測試通常要走 `localhost` 或 `127.0.0.1`。
7. 同一個業務通常會跨前端 view、api、controller、service、repository、entity、SQL 與通知層，建議用流程追程式，不要只按資料夾看。
8. PDF 中文顯示目前共用 `PdfCjkFontLoader` 與 `NotoSansTC-Regular.ttf`，若新增 PDF 服務，建議沿用相同工具，避免正式環境找不到本機字型。
